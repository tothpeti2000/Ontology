Hatalmas érdeklõdés közepette rajtolt el Magyarországon a Windows Server 2008

Magyarországon ezekben a percekben indítja útjára következõ generációs Windows szerverét a Microsoft. A Windows Server 2008 hazai bejelentését láthatóan rendkívül nagy érdeklõdés övezi, a résztvevõk állva és a lépcsõkön ülve, rossz szóviccel élve mondhatni hõsiesen hallgatják a nyitó elõadásokat, miután színültig telt a Palace Lurdy legnagyobb terme.



Alapos munka

A globális amerikai rajtot követõen ma Magyarországon is elindult hivatalosan a Windows Server 2008, valamint a zászlóshajóval egyetemben a SQL Server 2008 adatbáziskezelõt és Visual Studio 2008 fejesztõkörnyezetet is bejelentették. Gál Tamás, a Microsoft Magyarország Windows 2008 szakmai vezetõje szerint az 5 év fejlesztést maga mögött tudó Windows Server 2008 rendkívül alapos, nagyjából két évnyi nyúzást követõen kerül élesítésre, a bétaprogramban rengetegen vettek részt, és a bétatesztelés igen pörgõsnek bizonyult. A cég bejelentése szerint az elõzetesek kipróbálásával, észrevételeivel, visszajelzéseivel világszerte összesen 3 millió vállalat kapcsolódott be a termékek fejlesztésébe.

Gál elmondta, a rengeteg újdonságot felvonultató új szerverplatform  a Vista SP1-gyel közös kódbázisra alapoz, vagyis már gyárilag tartalmazza azokat a frissítéseket -- a WS08 névjegye is egyes szervizcsomagot mutat a gyári telepítést követõen. A lokalizált változat március 19-én kerül gyártásba, és április elsejétõl hozzáférhetõ volumenlicencelési konstrukcióban. A nyitó elõadáson megjelent ügyfélreferenciák alapján már itthon is több éles rendszer üzemel Windows Server 2008 alapokon, köztük például a freeblog.hu is, mely érdekes módon egyetlen kétmagos Opteron processzorral szolgálja a napi több tízezer látogatót és több százezer lekérést -- a statikus fájllekérésekkel együtt elhangzott a milliós nagyságrend is. A Visual Studio 2008 szintén hamarosan elérhetõvé válik, az SQL Server 2008-ra azonban még néhány hónapot várni kell.

A hosszadalmas, publikus tesztelési fázisból következik, hogy a WS08 régóta ismert, így maga a bejelentés technikai oldalról már igen nehezen szolgálhat újdonsággal -- ha mégis tartogatott volna valami váratlan meglepetést a Microsoft, arra valószínûleg a múlt heti Los Angeles-i rajton fény derült volna. Az új szerver Windows nyolcféle változatban kapható majd: a Standard, Enterprise és Datacenter Hyper-V virtualizációval, vagy anélkül hatot tesz ki, ezenkívül van még Web Server és Itanium Edition is. Frissítési útvonalat a WS03 Standard változatok számára a WS08 Standard és Enterprise, a WS03 Enterprise számára a WS08 Enterprise, míg a Datacenter számára az új Datacenter szolgáltat. A WS03 Web Server és az Itanium Edition nem frissíthetõ, ott migráció szükséges.

Bizalomépítés a biztonságon keresztül

Ahogyan az amerikai bejelentés alkalmával, úgy a budapesti nyitóbeszédekben is hangsúlyozták, hogy a WS08 fejlesztéseinek fókuszát a kiemelten a biztonság és rendelkezésre állás képezte. Mint elhangzott, a Microsoft szoftvereiket a múltban sok kritika érte a sebezhetõségek miatt, ami nagyrészt azok elterjedtsége magyaráz: a támadók és a média figyelme egyaránt a cég termékeire irányulnak, így a biztonsági rések és hibák mindig gyorsan és nagy publicitást nyertek.

A biztonság szem elõtt tartása már a telepítésnél megkezdõdik, mikor azokat a komponenseket installálja csak a rendszer, melyekre szerepe alapján szüksége lesz -- ennek az elvnek egyik szélsõséges formája a Server Core telepítés, mikor alig néhány alapszolgáltatás (DNS, DHCP, tartományvezérlõ, fájlszerver) kerül a rendszerbe. A csak parancssorból vezérelhetõ Server Core nem csak a támadási felületet minimalizálja, de gépigénye is igen szerény, ráadásul a kevés összetevõ miatt a hiba lehetõsége és a frissítés gyakorisága is csökken -- vagyis egyszerû és olcsó üzemeltetni.

[+] A Windows Server 2008 újdonságai

További biztonsági funkciók, mint a csak olvasható tartományvezérlõ (RODC), vagy a belsõ hálózatot a potenciálisan veszélyt jelentõ gépekkel szemben védõ Network Access Protection annak felismeréseként született, hogy az IT-rendszereket nemcsak a határvonalon, a külsõ és belsõ hálózat határánál kell védeni, hanem a belülrõl érkezõ támadásokkal szemben is -- megbukott a lakópark modell, ahogyan a Microsoft fogalmaz. A WS08 ezenkívül az IIS7 révén nagyteljesítményû PHP-, Perl- és Ruby-futtatási környezetet, egyszerûbb felügyeletet és fejlett virtualizációs képességeket is ígér, melyekkel teret nyerhet a Linux disztribúciókkal szemben is, ami a jelenlegi hazai tendenciák alapján kifejezetten igaz lehet Magyarországon. 

A CDW által nemrég megkérdezett mintegy 800 informatikai döntéshozó 49 százaléka szerint a megnövekedett biztonság a WS08 legnagyobb vonzereje, ezen kívül a legvonzóbb tulajdonságok a gyorsabb és egyszerûbb telepítés és konfigurálás (41%), a könnyebb adminisztráció (40%). A megkérdezettek 48 százaléka ugyanakkor tart a szoftverhibáktól, 25 pedig az alkalmazásai támogatottságáért aggódik; 19 százalék számára pedig nem világosak a Windows Server 2008 várható elõnyei. A válaszok alapján a szervezetek 63 százaléka tervezi valamikor a jövõben bevezetni a Windows Server 2008-at, de a megkérdezettek 18 százalékánál már meg is kezdõdött az átállás, vagy legalábbis a migráció tervezése.
