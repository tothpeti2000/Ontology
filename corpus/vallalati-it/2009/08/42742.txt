Letölthetõ és kipróbálható a Windows Server 2008 R2

A Windows 7 után a Windows Server 2008 R2 végleges kódját is elérhetik már azok, akik MSDN- vagy TechNet-elõfizetéssel rendelkeznek, illetve a mennyiségi licencvásárló ügyfelek is letölthetik az új szerver operációs rendszert, de egyelõre csak angolul, a lokalizált változatokra októberig várni kell. A Windows Server 2008 R2 végleges változatának kiadásával együtt a Microsoft elérhetõvé tette a termék próbaváltozatát is, a 180 napig korlátozás nélkül használható szoftver összes változata (Web, Standard, Enterprise, Datacenter, Itanium) telepítõkészletként és virtuális lemezképként is letölthetõ a Microsoft weboldaláról.
A legfontosabb újdonságok
Emlékeztetõül: a Windows Server 2008 R2 az elsõ Microsoft operációs rendszer, amelybõl kizárólag 64 bites változat jelenik meg, így használatához mindenképp 64 bites processzorra (AMD64, Intel 64) épülõ konfigurációra van szükség. Az Opteronok és Xeonok már évek óta támogatják a 64 bites végrehajtást. A 32 bites alkalmazások persze továbbra is futnak a 64 bites Windows Serveren.
A Microsoft azt ígéri, a chipekben található legújabb utasításkészletek támogatása révén a Windows Server 2008 R2 minden eddiginél nagyobb teljesítményt nyújt majd, a kód optimalizálásának köszönhetõen akár korábban megvásárolt szerverek is "megtáltosodnak". Az operációs rendszer legfeljebb 256 logikai processzort támogat, így hatalmas szerverekre is feltelepíthetõ. A cég szerint a skálázódás nem csak papíron mûködik, a mérések szerint egy 128 magos rendszer SQL Serverrel 1,7-szer nagyobb tranzakciófeldolgozási sebességre képes mint egy 64 magos ugyanazzal az adatbázissal.
Az erõforrásgazdálkodás terén is sokat lép elõre az operációs rendszer a processzorok energiatakarékos technológiáinak még jobb kihasználásával: automatikusan képes a processzorok éppen nem használt magjait lekapcsolni, ha nincs rájuk szükség. Az adminisztrátorok akár csoportházirendbõl is szabályozhatják majd, hogy a kiszolgálók mely idõszakokban kapcsoljanak energiatakarékos módba. A Microsoft szerint a Windows Server 2008-hoz képest jelentõsen nõtt az energiahatékonyság, de még nagyobb a különbség a még mindig széles körben használt Windows Server 2003-hoz képest. Az új szerver operációs rendszer jóval kevesebb memóriát is igényel az elõdöknél, így több jut az alkalmazásoknak.
A Windows Server 2008 R2-t szorosan integrálták a PowerShell 2.0 parancssorral, így az adminisztrátorok az operációs rendszer legutolsó bitjét is képesek lesznek parancssorból elérni. Az adminok életét megkönnyítendõ az R2-be a Microsoft több mint 200 elõre megírt PowerShell-szkriptet is épít a leggyakrabban használt szerepkörök (pl. AD, DHCP, DNS) felügyeleti feladatainak automatizálására. A "csupasz" Windows Server Core-t is továbbfejlesztik, az R2 változatban így telepített operációs rendszer is támogatja az ASP .NET-et, illetve a PowerShellt (ennek elõfeltétele a .NET-környezet megléte).
Windows Server 2008 R2 és Windows 7
Ahogy a Vistában és a Windows Server 2008-ban számos olyan képesség volt, amiket csak együtt lehetett kihasználni, nem lesz ez másként a Windows Server 2008 R2 és a Windows 7 esetében sem. Az egyik ilyen újdonság a DirectAccess, ami elsõsorban a virtuális magánhálózaton (VPN) keresztül kapcsolódó alkalmazottak életét könnyítheti meg. Windows 7 és Windows Server 2008 R2 együttes használata esetén nem lesz szükség VPN-re, a felhasználók az interneten keresztül, IPv6 és IPSec használatával ugyanúgy elérhetik a belsõ hálózatot, mintha ott ülnének az irodában.

Szintén a telephelytõl távol, például egy fiókirodában dolgozó felhasználók életét könnyíti meg a Branch Cache, ami a helyi gépre gyorsítótárazza az SMB-n és HTTP-n letöltött tartalmakat - ha nincs helyi szerver, a gyorsítótár akár egy Windows 7-et futtató PC is lehet. A Windows 7 korábbinál fejlettebb biztonsági és energiatakarékossági képességei megjelennek a Windows Server 2008 R2 csoportházirend-objektumai között is, így a rendszergazdák még finomabban állíthatják be azokat, például a hordozható eszközökön tárolt adatok titkosítását a BitLockerrel.
A korábbi Terminal Services jelentõsen továbbfejlõdött, amit névváltás is hangsúlyoz, mostantól Remote Desktop Servicesnek (RDS) hívják. Az RDS egyik legfontosabb újdonsága a Virtual Desktop Infrastructure (VDI), ami a korábban ismert "távoli asztal" megoldás mellett már lehetõvé teszi a windowsos asztali környezetek virtuális gépként való, egységes kezelését az adatközpontban. A RemoteApp és Desktop kapcsolatok integrációja a Windows 7-tel azt eredményezi, hogy az új asztali operációs rendszer felhasználóinak egyátalán nem kell azzal törõdniük, mely alkalmazások távoliak és melyikek helyiek - egyszerûen csak el kell indítaniuk õket a Start menübõl.
Az RDS olyan újdonságokat is felvonultat, mint a továbbfejlesztett többmonitoros támogatás, illetve a jobb audio- és videoképességek, amelyek révén lehetõvé válik akár a komoly multimédiás teljesítményt igénylõ alkalmazások központosítása is. A Calista felvásárlásával a Microsoft kezébe került vGPU technológia ugyanis lehetõvé teszi, hogy a 3D megjelenítés a szerverbe épített grafikus gyorsító feladata legyen, a kliensek csak a megjelenítést végzik. Így lényegében olyan multimédiás alkalmazások is bevonhatók egy virtualizált desktop infrastruktúrába, amelyeket korábban csak helyben lehetett futtatni.
További újdonság, amelyet a Windows 7-et és Windows Server 2008 R2-t használók élvezhetnek, a megnövekedett fájlmásolási sebesség WAN-on keresztül. A Microsoft azt állítja, a kis és közepes méretû állományok feltöltése 20 százalékkal gyorsabb mint Windows Server 2008 és Vista használata esetén, a letöltés pedig 47 százalékkal gyorsult, a nagy fájlok mozgatása pedig fele annyi idõt vesz igénybe mint korábban. Az új, többszálúságra felkészített RoboCopynak köszönhetõen a WAN-ok közötti fájlmásolás akár nyolcszor is gyorsabb lehet mint korábban.
Virtualizáció
Virtualizáció területén is sokat lépett elõre a Windows Server 2008 R2, ugyanis a Hyper-V köztesréteg 2.0 változatát tartalmazza már, ami azt jelenti, hogy végre támogatott a hibatûrõ fürtök építése, valamint a virtuális gépek leállítás nélküli mozgatása a szerverek között. Ezeket a képességeket a VMware már egy ideje kínálja, azonban a Microsoft Hyper-V nem feláras extraként, hanem alapfunkcióként nyújtja õket.
További újdonság az RC-ben a VM Chimney, ami lehetõvé teszi a virtuális gépek összerendelését a hálózati kártyákkal, kihagyva a virtuális hálózati interfészt, aminek az eredménye nagyobb hálózati teljesítmény olyan feladatok alatt mint például az SQL mentés és visszaállítás, vagy éppen virtuális gépek mozgatása. Ez a szolgáltatás alapértelmezésként azonban le van tilva, mivel egyes, nem szokványos feladatok futását lassíthatja. Egy másik új fejlesztés a Virtual Machine Queue (VMQ) támogatása, ez a hálózatkezeléssel kapcsolatos egyes feladatokat veszi le a hypervisor válláról és a hálózati vezérlõre terheli át. Ehhez megfelelõ hálózati vezérlõre is szükség van.
