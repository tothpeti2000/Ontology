Itt az Exchange Server 2010 kiadásra jelölt változata

Elérhetõ a Microsoft Exchange Server 2010 véglegesnek szánt (RC) változata, a szoftvert bárki letöltheti a cég weboldaláról. Ez az elsõ olyan elõzetese a csoportmunka- és levelezõszervernek, amely a végleges verzióba szánt összes képességet, szolgáltatást tartalmazza. A termék a tervek szerint az õsszel kerül kereskedelmi forgalomba, alighanem a Windows 7-tel és a Windows Server 2008 R2-vel együtt.
Újdonságok az RC-ben
Az eddig kiadott elõzetesekhez képest több újdonság is bekerült a megjelenésre szánt verzióba. Ezek közül az egyik legfontosabb, hogy az RC már támogatja a 64 bites Windows Server 2008-at és a Windows Server 2008 R2-t, vagyis az õsszel érkezõ új szerverre is telepíthetõ. A tesztelést és bevezetést könnyíti, hogy a Release Candidate verzió lehetõséget ad a felhasználóknak arra, hogy újratelepítés nélkül, helyben frissítsék az Exchange Server 2010-et a végleges változatra. Szintén a tesztelés és bevezetés felgyorsítása érdekében az új Exchange használható olyan rendszerekben, ahol már üzemel Exchange Server 2003 vagy 2007, bár az utóbbival való együttmûködéshez az Exchange Server 2007 SP2 telepítése szükséges. Ez az õsszel fog megjelenni.
A Microsoft az új Exchange mellett több olyan szoftver is kiadott, amelyek a bevezetést és a migrációt hivatottak támogatni. Ezek közé tartozik a System Center Operations Manager 2007 R2-vel használható felügyeleti csomag, illetve olyan megoldások, amelyekkel az Exchange még éles üzembe állítás elõtt tesztelhetõ: a távoli kapcsolatok tesztelésére szolgáló Exchange Server Remote Connectivity Analyzer, a diszkterhelési tesztet végzõ Exchange Server JetStress Tool for Exchange Server 2010, illetve az Exchange Load Generator (LoadGen), amellyel az egész rendszert lehet terheléses tesztnek alávetni különféle kapcsolatok (MAPI, IMAP, POP, SMTP, OWA) szimulálásával. Utóbbi két eszköz segítségével megállapítható, hány Exchange szerver szükséges adott terhelés mellett.
Gyorsabb
Az Exchange Server 2010 a Microsoft ígérete szerint gyorsabb lesz elõdeinél, többet tud azoknál és mégis könnyebben felügyelhetõ, karbantartható. Az Exchange továbbra is a Jet adatbázismotort használja, pedig ennek lecserélése az SQL Serverre már évekkel ezelõtt szóba került. A Microsoft mérnökei azonban idõközben annyira felturbózták a Jetet, hogy végül szükségtelenné vált a csere. a korábbiaknál egyenletesebben írja és olvassa az adatokat, ennek eredményeképp az I/O sebesség akár 70 százalékkal nagyobb lehet az Exchange Server 2007-énél és közel kétszeresen múlja felül az Exchange Server 2003-at. A postafiókok kezelésében végrehajtott változtatások lehetõvé teszik, hogy egy mappa (folder) akár 100 ezer levelet is tartalmazhasson, szemben a korábbi 20 ezres korláttal.
Az Exchange Server 2010 továbbra is öt szerepkört vehet fel, fontos változás azonban hogy a Mailbox Serverrõl több funkció átkerült a Client Access Serverre, ennek köszönhetõen a Mailbox Server akár 120 ezer egyidejû kapcsolat kiszolgálását is végezheti, tízszer annyit mint az Exchange Server 2007-ben. Meghibásodás esetén a Mailbox Serverek most már 30 másodperc alatt képesek átterhelõdni másik gépre, a korábbi Exchange esetében ez két percet vett igénybe legalább.

A nagyobb teljesítményt a hardverigényen kell megfizetni. A Client Access Serverként üzemelõ Exchange-szerverek legalább 8 processzormagot és 8, de inkább 16 gigbájt memóriát igényelnek majd.A kisebb cégek akár egyetlen szerverre is tehetik az összes Exchange Server szerepkört, de a Microsoft inkább a horizontális skálázást ajánlja - a lehetõség mindenesetre adott akár 16 processzormag használatára is.
A Microsoft új üzenetkezelõ szerverének fejlesztõi nagy hangsúlyt fektettek a megbízhatóságra és rendelkezésre állásra, valamint olyan új fürtözési képességeket is bevezettek, amelyre alapozva a korábbi megoldások (CCR, LCR, SCR, SCC) elõnyeinek ötvözésével egy "bombabiztos" fürt hozható létre már két szerverbõl.
Okosabb
Az Exchange Server 2010 a korábbinál fejlettebb képességeket vonultat fel az üzenetek megõrzése és archiválása terén, a szerver minden postafiókot és mappát elõre beállított szabályrendszerek alapján tud majd kezelni. Érdekes újdonság, hogy bizonyos típusú leveleket az Exchange Server 2010 a Microsoft Rights Management Services (RMS) segítségével automatikusan, a küldõ szándékától függetlenül képes jogvédelemmel ellátni, ezzel csökkentve a bizalmas információk kiszivárgásának kockázatát. Szintén az információszivárgást megakadályozni hivatott fejlesztés, hogy a szoftver automatikusan figyelmezteti a felhasználót, ha túl nagy csatolmányt akar küldeni egy terjesztési listára.
A vállalaton belüli felesleges e-mail forgalom csökkentésére is tartalmaz néhány új trükköt az új Exchange Server. A szoftver levélküldés elõtt képes figyelmeztetni a felhasználókat, ha olyasvalakinek akarnak e-mailt küldeni, aki éppen nincs az irodában (például be van állítva az out of office autoreply), megtelt a postafiókja vagy például nem tudja a csatolt állományt fogadni. A MailTips nevû fejlesztés nem csak erre képes, segítségével megakadályozható a hirtelen felindulásból írt levelek elküdése is, ha pedig valaki mégis rákattintott a "küldés" gombra, egy pár másodpercig visszavonhatja azt. A hatékonyabb levelezés érdekében a szoftver a Gmailhez hasonlóan képes az üzeneteket "beszélgetésekbe" szervezni, amelyeket akár el is lehet némítani.
Megújul az Outlook Web Access is, amely az Outlook Live nevet kapta. A böngészõbõl elérhetõ felület megszólalásig hasonlít a telepített Outlookra és egyre több olyan szolgáltatás érhetõ már el a segítségével, amelyek eddig csak a telepített változatban voltak jelen. Az Exchange Server 2010-ben változatban ilyen például a terjesztési listák kezelése, szabályok létrehozása, vagy más postafiókok megnyitása. Jó hír, hogy az OWA már nem csak Internet Explorerbõl, hanem Firefoxból és Safariból is használható, rossz hír, hogy az újdonságok egy része (pl. MailTips) Windows Mobile-ról nem lesz elérhetõ.
Felügyelet
A rendszergazdák, adminisztrátorok a korábbiaknál fejlettebb felügyeleti funkcióknak örülhetnek majd. Egy, a Microsoft megbízásából készült tanulmány azt állítja, az Exchange Server 2007 üzemeltetési költsége mintegy fele az Exchange 2003-énak és az Exchange Server 2010 esetében hasonló mértékû csökkenés várható. A rendszergazdák életét olyan újdonságok hivatottak könnyebbé tenni, mint a távfelügyelet PowerShell segítségével, a szerep-alapú hozzáféréskezelés (Role Based Access Control) integráció, amely jogosultságokhoz köti a PowerShell parancsok futtatását, illetve az adminisztrátori tevékenység naplózása, amellyel visszakövethetõvé és ellenõrizhetõvé válik a rendszergazdák tevékenysége.
Kellene az Office 2010 is
Bár az Exchange Server 2010 megjelenésre szánt változata funkciókész, a felhasználói oldalon megjelenõ újdonságok teszteléséhez szükség van az Outlook 2010-es változatára, amely az Office 2010 részeként jelenik meg és jelenleg csak zárt béta formájában érhetõ el, vagy fájlcserélõ hálózatokról lehet letölteni. Az Exchange Server 2010 újdonságairól bõvebben a Microsoft TechNet oldalán lehet olvasni.
