Kipróbálható a SQL Server 2008 R2 elsõ elõzetese

A Microsoft szervertermékeibõl négyévente jelenik meg teljesen új kiadás, kétévente pedig ezek kisebb frissítése. Az egy éve kiadott SQL Server 2008-at jövõre frissíti a Microsoft a Kilimanjaro kódnevû R2 verzióval, amelynek újdonságai közül néhány már kipróbálható a most kiadott CTP-ben. A végleges változatot 2010 elsõ felére ígéri a Microsoft.
Újdonságok az augusztusi CTP-ben
A SQL Server 2008 R2 elsõ technikai elõzetese két fontos új képesség kipróbálását teszi lehetõvé. Ezek közül az egyik az "Application and Multi-Server Management" (AMM), azaz több alkalmazás és szerver központi felügyelete, amely lehetõvé teszi a rendszergazdák számára, hogy több SQL Serverbõl álló környezeteket központilag menedzseljenek, egyetlen panelen figyelemmel kísérve az egyes adatbázisok és alkalmazások teljesítményét és a szerverek terheltségét, kihasználtságát. Ezekhez szabályrendszereket, riasztásokat is lehet kapcsolni.
A SQL Server 2008 R2 bevezeti az ún. data-tier alkalmazás fogalmát, amely gyakorlatilag az adatbázist, táblákat, tárolt eljárásokat tartalmazza magában a Visual Studio projektsablonban annak érdekében, hogy az alkalmazás telepítéséhez szükséges összes információ és feltétel egy helyen elérhetõ legyen, ezáltal az alkalmazást a lehetõ leggyorsabban lehessen telepíteni vagy éppen frissíteni. Az új SQL Server egyébként a korábbiaknál sokkal jobban skálázható, akár 256 logikai processzort is képes kezelni, ami csak elsõre tûnik soknak, hiszen mire a szoftver megjelenik, az Intelnek már lesz nyolcmagos Xeon processzora.
Az augusztusi CTP-ben a másik fontos fejlesztés a jelentéskészítéssel kapcsolatos, az SQL Server 2008 R2-ben ugyanis már a Report Builder 3.0 változata kap helyet. A Report Builderrel az üzleti felhasználók maguk hozhatnak létre különféle jelentéseket, amelyeket grafikonokkal és más adatvizualizációs eszközökkel színesíthetnek. A Report Builder 3.0 már támogatja a földrajzi adatok megjelenítését is, így az üzleti adatokat akár térképekkel is lehet kombinálni. A Microsoft emellett azt ígéri, az új Report Builder lényegesen gyorsabb az elõdjénél.
Ami kimaradt
A SQL Server fejlesztésének fókuszában jelenleg az üzleti intelligencia áll. A Kilimanjarót ennek érdekében a Gemini fedõnevû projekt alatt olyan képességekkel vértezték fel, amely lehetõvé teszi a szoros integrációt a Microsoft Office család tagjaival. Az Office-felhasználók közvetlenül a számukra már ismert felületen érhetik el az SQL Serverben tárolt adatokat, és az Office-ba épített képességek segítségével lényegében saját üzletiintelligencia-alkalmazásokat, egyfajta mash-upokat hozhatnak létre, amelyeket a SharePoint révén akár meg is oszthatnak egymással.
Az Excel vitán felül a legszélesebb körben elterjedt táblázatkezelõ, ami az üzleti életben de facto szabvánnyá vált, egyben maga is egy üzletiintelligencia-eszköz. Kézenfekvõnek tûnik tehát, hogy ezt az Excelt az üzleti intelligencia front-endjeként használja a Microsoft. A Business Intelligence konferencián elhangzottak alapján azonban nem csak az Office-szal, hanem a Dynamics 2009 vállalatirányítási termékekkel is szorosan együttmûködhet majd az új SQL Server. A Microsoft reményei szerint ez lesz a kulcselõnye az üzletiintelligencia-piacon.

Üzleti intelligencia terén az SQL Server 2008 R2 figyelemre méltó újdonsága még a StreamInsight nevû technológia, amelyet kimondottan a valós idejû adatmonitorozásra fejlesztettek ki. Számos iparágban (pl. gyártás, pénzügy, kereskedelem) szükség lehet egy folyamatos adatáram elemzésére anélkül, hogy ezeket tárolni kellene. A StreamInsight lényegében az adatbányászatra és -elemzésre szolgáló lekérdezéseket engedi rá a valós idejû adatfolyamra.
A StreamInsight és a Gemini a most kiadott  augusztusi CTP-ben még nincs benne, de a Microsoft ígérete szerint a következõ hetekben újabb elõzetesek várhatók, így valószínûleg hamarosan ezek a képességek is tesztelhetõvé válnak. Szintén még az idén számíthatunk a Madison kódnevû szoftverre, amely egy kimondottan üzleti intelligencia feladatokra szánt SQL Server változat, amely a DATAllegro felvásárlásával megszerzett technológiákat is tartalmazza. A 2010 elejére várt Madisonnal állítólag több száz csomópontos BI-clusterek építhetõk, akár petabájtos kapacitással.
