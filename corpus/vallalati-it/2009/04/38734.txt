Szerveroldali alkalmazásvirtualizáció és élõ migráció a Microsofttól is

Az egyik legnagyobb újítás, hogy a Microsoft Application Virtualization, vagyis az App-V funkcionalitását kiterjesztik a szerveroldali alkalmazásokra is. Ez annyit tesz, hogy az olyan szerveralkalmazások, mint például az SQL Server  vagy Exchange, kapnak egy sandboxot, akár más, szorosan kapcsolódó alkalmazásokkal közösen. Ez lényegében a Microsoft megoldása a virtuális célgépekre, vagyis a virtual appliance-re -- igaz, az operációs rendszert továbbra sem lehet nélkülözni, a Microsoft "csomagjai" nem futnak közvetlenül a hardveren. A virtual appliance egy olyan virtuális gép, melyet egy adott alkalmazás kiszolgálására készítettek fel. Eddig hasonló megoldás Linux-alapokon létezett, ahol egy minimális, kizárólag a szükséges modulokat és szolgáltatásokat kínáló operációs rendszermag szolgálta ki a rátelepített és felkonfigurált alkalmazásokat.

Az így virtualizált, specifikus virtuális alkalmazásképek könnyedén sokszorosíthatóak és mozgathatók, továbbá az operációs rendszerek képét is leegyszerûsíti, hiszen az alkalmazás többé nem integrálódik az operációs rendszerrel. Ez a megoldás desktopkörnyezetben is rendkívül hasznosnak bizonyulhat, a Microsoft App-V képes a becsomagolt alkalmazást egy szerverrõl áttölteni a kliensre, anélkül, hogy telepíteni kellene bármit is. Hasonló képességgel bír a VMware ThinApp is, mely nem csak virtualizált, távoli desktopkörnyezetben, de lokális fizikai gépeknél is hasznos, mivel ugyanúgy függetleníti az alkalmazásokat az operációs rendszertõl.


Ahogyan azt már korábban a HWSW is megírta, a Windows Server 2008 R2-vel a Hpyer-V képessé válik a Live Migrationre, vagyis a virtuális gépek leállítás nélküli mozgatására. Ez a képesség kiemelten fontos a magas rendelkezésre állást megkövetelõ rendszerek esetében, ugyanis a hardver vagy a VM alatt meghúzódó szoftverréteg karbantartása vagy problémás mûködése esetén sem kell egy pillanatra sem leállítani a VM-et. Ez a funkció a VMware vMotion megfelelõje, de mozgatásra képes a Xen és a KVM virtualizációs technológia is.

Ezek a virtualizált alkalmazáscsomagok és az élõ migráció olyan képességek, melyek alapvetõek a cloud computing víziójának megvalósításában. A Microsoft egyúttal ismét leszögezte, lehetõvé kívánja tenni a magán- és nyilvános felhõk közötti átjárást, akár automatizált, politikákkal vezérelt szabályok mentén, amire felkészítik majd a System Center Virtual Machine Managert. A Microsoft elképzelése szerint a szervezet saját rendszerének terheltsége függvényében lehetne hosting partnerek vagy a Microsoft Azure felhõjébe átmozgatni a feladatok futtatását, ahol azt automatizált módon mozgatják a megfelelõ gazdagépre -- vagy éppenséggel visszaküldeni azt a szervezet saját gépére.

A fajsúlyosabb bejelentések közé tartozik még a System Center Online felügyeleti család elsõ eleme, a Desktop Manager. A Desktop Manager lényegében egy távoli felügyeleti megoldás, melyet két év éles környezetekben történõ tesztelést követõen jelent be kereskedelmi termékként a Microsoft. Az eszköz a szokásos távoli menedzsment funkciókon (távoli vezérlés, patchelések, konfigurálás) felül többek közt nyilvántartást vezet a hardverekrõl és szoftverekrõl, jelentéseket készít a hibákról. A szolgáltatás átveszi a Windows Update helyét egy telepítendõ frissítést követõen,  pontosabban átirányítja azt a System Center Online szerverére. A rendszergazdák akár csoportokban is kezelhetik a PC-ket, a jövõben pedig a szerverekre is kiterjed majd ez a felügyeleti képesség.
