Látványos gyorsulást ígér az Exchange Server 2010

Az Exchange Server 2010 továbbra is a Jet adatbázismotort használja, pedig ennek lecserélése az SQL Serverre már évekkel ezelõtt szóba került. A Microsoft mérnökei azonban idõközben annyira felturbózták a Jetet, hogy végül szükségtelenné vált a csere.
Gyorsabb
A TechEden tegnap elhangzottak szerint az Exchange Server 2010 a korábbiaknál egyenletesebben írja és olvassa az adatokat, ennek eredményeképp az I/O sebesség akár 70 százalékkal nagyobb lehet az Exchange Server 2007-énél és közel kétszeresen múlja felül az Exchange Server 2003-at.A postafiókok kezelésében végrehajtott változtatások lehetõvé teszik, hogy egy mappa (folder) akár 100 ezer levelet is tartalmazhasson, szemben a korábbi 20 ezres korláttal.&nbsp;
Az Exchange Server 2010 továbbra is öt szerepkört vehet fel, fontos változás azonban hogy a Mailbox Serverrõl több funkció átkerült a Client Access Serverre, ennek köszönhetõen a Mailbox Server akár 120 ezer egyidejû kapcsolat kiszolgálását is végezheti, tízszer annyit mint az Exchange Server 2007-ben. Meghibásodás esetén a Mailbox Serverek most már 30 másodperc alatt képesek átterhelõdni másik gépre, a korábbi Exchange esetében ez két percet vett igénybe legalább.
A nagyobb teljesítmény ára nagyobb hardverigény. A Client Access Serverként üzemelõ Exchange-szerverek legalább 8 processzormagot és 8, de inkább 16 gigbájt memóriát igényelnek majd.A kisebb cégek akár egyetlen szerverre is tehetik az összes Exchange Server szerepkört, de a Microsoft inkább a horizontális skálázást ajánlja -- a lehetõség mindenesetre adott akár 16 processzormag használatára is.

kerül, alighanem egyszerre mutatják be a Windows 7-tel és a Windows

Az Exchange Server 2007 SP2 kövezi ki az utat a 2010 számára
Az Exchange Server 2010-zel kapcsolatos újdonságok mellett a Microsoft ismertette, milyen funkciók várhatók az&nbsp; Exchange Server 2007 õsszel érkezõ második javítócsomagjában. A redmondi cég szerint az Exchange Server 2007 SP2 megkönnyíti majd az átállást Exchange Server 2010-re, ezért mindenképp ajánlott a telepítése azoknak a felhasználóknak, akik késõbb a 2010-es verzióra szeretnének migrálni. A 2010-es verzióval való együttmûködéshez nélkülözhetetlen is a 2007 SP2, enélkül a két termék nem tud szót érteni egymással.

Az SP2-be bekerültek az auditálást megkönnyítõ fejlesztések, megjelent egy külön tárház (repository) az audit logok számára. A rendszergazdák életét teszi kényelmesebbé az SP2-ben debütáló új PowerShell programok, amelyekkel az Exchange Server szervezeti beállításait lehet módosítani, illetve a kvótákat is be lehet állítani. További újdonság, hogy az Exchange Management konzolból már diagnosztikai logokat is el lehet érni. A Volume Snapshot Backup segítségével a Windows Server 2008-ból indított mentésekkel együtt menthetõ az Exchange is, ezt a szolgáltatást eddig csak utólag telepített bõvítményekkel lehetett elérni.
Az egyik legérdekesebb újdonság az Active Directory sémák kezelésével kapcsolatos. Az Exchange Server 2007 SP2-ben várhatóan megjelenik az automatikus sémabõvítés, vagyis ha az Active Directory olyan objektumtípusra vagy -tulajdonságra hivatkozik, amely nincs benne a sémában, akkor az Exchange Server automatikusan bõvíti a sémát.
