A Microsoft a PHP-t is szeretné felhõjében látni

A Microsoft a PHP-val történõ együttmûködést segítõ eszközöket a nyílt forráskódú fejlesztéseit összegyûjtõ CodePlexen tette közzé. A PHPAzure SDK lényegében lehetõvé teszi, hogy a Microsoft növekvõben lévõ felhõjébe helyezzünk PHP-s alkalmazásokat, így azok képesek lesznek az Azure-ral REST (Representation State Transfer) és XML felületeken keresztül kommunikálni, prózai módon fogalmazva hozzáférve a számítási és tárkapacitáshoz, valamint a menedzsment funkciókhoz. 
A PHP SDK-val felkészített alkalmazás képes lesz akár kívülrõl, vagyis egy internetre csatlakozó szerverrõl is elérni az Azure erõforrásait, a PHP motornak nem kell a Microsoft felhõjében futnia. A Microsoft hasonló, az interoperabilitást célzó SDK-t már kiadott Java és Ruby nyelvekhez is. Ezek az SDK-k lényegében a hívások megfelelõségét biztosító osztályokat kínálják. A PHPAzure SDK BSD licenccel rendelkezik jelenleg.

A Microsoft továbbá olyan nyílt forrású kódot és projektet is közzétett a CodePlexen, melyekkel a PHP-fejlesztõk Microsoft technológiákat használhatnak fel, mint például a Silverlight tartalmak és vezérlések, vagy az Internet Explorer 8 WebSlices kezelésének elsajátítása. A CodePlex hivatalosan 2006 nyarán indult el azzal a célzattal, hogy teret nyisson a Microsoft valamint a cég platformjaihoz közeli, nyílt forrású fejlesztéseknek. Jelenleg több mint 8 ezer projektet számlál.
Az Azure a Microsoft cloudszolgáltatása, a mögötte meghúzódó platform kereskedelmi termékként nem lesz elérhetõ. A vállalat szerint Windows Serveren alapul ugyan, ugyanakkor nem volna alkalmas az értékesítésre, mivel a Microsoft adatközpontjaival integrálódik. A Microsoft célja a Windows Azure-rel, hogy egy olyan platformot hozzon
létre, amelyen a Windowsra írt alkalmazások ugyanúgy futnak mintha csak
egy Windows Serverre telepítették volna õket. A PHP támogatásával az eddig az ASP.NET és .NET, Java és Ruby nyelveken álló alakalmazások tárháza bõvült.
