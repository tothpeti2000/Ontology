Támogatottak a kevert RHEL és Windows Server virtuális környzetek

A Microsoft és a Red Hat februárban jelentették be, hogy megállapodást kötöttek a két cég szervervirtualizációs termékeinek együttmûködésének biztosítására, valamint hogy ezekhez a heterogén rendszerekhez mûszaki támogatást is fognak kínálni a jövõben. Mindez azt jelenti, hogy mind a cégek garantálják olyan rendszerek megbízható mûködését és támogatását, amelyek Red Hat Enterprise Linux gazdákon Windows Server vendégeket fogadnak, vagy fordítva, Windows Server vagy Hyper-V Server rétegen RHEL VM-eket.
A felek most közölték, hogy a megállapodást követõen 8 hónappal sikeresen befejezõdtek a tesztelések, így már az ügyfelek támogatott heterogén virtuális környezeteket építhetnek a két cég operációs rendszereibõl. Fontos részlet, hogy üzletpolitikai okokból a Red Hat kizárólag a friss, ropogós RHEL 5.4 kiadást a KVM hypervisorral minõsítette támogatott gazdának, így ebbõl a megállapodásból kimaradnak a Xen hypervisort alkalmazók. Azt is érdemes tudni, hogy a cégek menedzsmentalkalmazásai egyelõre nem állnak készen arra, hogy egy heterogén virtuális környezeteket kezeljenek, de itt is gyors továbblépés várható a következõ hónapok során.

Az RHEL 5.4 KVM hoszton Windows Server 2003, 2008 és Windows Server 2008 R2 operációs rendszerek futhatnak. A másik oldalról némileg gazdagabbak az opciók, a minõsített gazdák közt a Windows Server 2008 Hyper-V, Microsoft Hyper-V Server 2008, Windows Server 2008 R2 Hyper-V és Microsoft Hyper-V Server 2008 R2 támogatottak, míg a vendégek RHEL 5.2, 5.3 és 5.4 platformok lehetnek - a Server 2003 azonban itt sem támogatott, mint virtualizációs alap. Ezzel a két cég termékeivel kapcsolatban megoldódik a heterogén virtualizált rendszerek egyik rákfenéje, a beszállítók egymásra történõ ujjal mutogatása.
További hatalmas elõnye az együttmûködésnek, hogy nem csak az operációs rendszerek támogatása oldódik ezzel meg, hanem az egyéb Microsoft vagy Red Hat szoftvereké is, melyek ilyen heterogén virtualizált környezetben üzemelnek. Ez azt jelenti, hogy egy RHEL 5.4 KVM hoszt feletti Windows Server VM-ben futó SQL Server ugyanazt a támogatást kaphatja meg ezentúl, mintha egy teljesen Microsoft környezetben mûködne - és természetesen fordítva, például egy Hyper-V Serveren lévõ RHEL 5.3 VM-ben futó JBoss is támogatott lehet. A támogatottság kérdése ugyanakkor természetesen attól is függ, hogy virtualizált környezetben támogatott-e a szoftver.
